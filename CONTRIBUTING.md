## 💡 Ways to Contribute

This is a data repository — the proxy files are generated and published automatically by our private engine. You cannot contribute proxy sources or scraper code directly. However, there are meaningful ways you can help:

### 🐛 Report Dead or Misbehaving Proxies
If you find that proxies from this repo are consistently dead, returning incorrect responses, or behaving unexpectedly, open an issue and include:

- The proxy address (e.g. `192.0.2.1:8080`)
- The type (HTTP / SOCKS4 / SOCKS5)
- What you expected vs. what happened
- The timestamp of the `files/timestamp.json` at the time of your test

### 📝 Improve Documentation
Spotted a typo, unclear wording, or a missing usage example? Pull requests that improve the `README.md` or any other markdown file are always welcome.

To make a documentation change:

1. **Fork** this repository
2. Edit the relevant markdown file
3. Open a **Pull Request** with a short description of what you changed and why

### 🌐 Translate Documentation
If you'd like to translate the README or legal documents into another language, open an issue first to coordinate so we don't duplicate effort.

### ⭐ Star the Repo
The simplest contribution — starring the repo helps others discover it and signals the project is active.

## 🐞 Reporting Issues

Before opening a new issue, please:

1. **Search existing issues** to avoid duplicates
2. Use a **clear, descriptive title**
3. Include **as much context as possible** — version of the timestamp file, language/library you're using, error messages

Open an issue here:  
👉 **[github.com/Casper-Tech-ke/casper-free-proxies/issues](https://github.com/Casper-Tech-ke/casper-free-proxies/issues)**

## 🚀 Suggesting Improvements

Have an idea for a new file format, endpoint, or feature? We'd love to hear it. Open an issue tagged `enhancement` with:

- A clear description of the feature
- Why it would be useful to the community
- Any examples or references you can share

## 🤝 Code of Conduct

This project follows a simple code of conduct:

- **Be respectful.** Treat everyone — contributors, maintainers, and users — with courtesy and professionalism.
- **Be constructive.** Criticism is welcome; hostility is not.
- **Be honest.** Report issues accurately. Don't exaggerate or fabricate problems.
- **Be patient.** This is a free community project. Response times may vary.

Violations may result in your contributions or issues being closed without response.
